@magic

magic is a module/component based generator for static pages and serverless deployments.

goal of the project is to keep the client boilerplate footprint minimal yet easily extendable

@magic-modules

the building blocks of @magic are modules that provide components.

  • @magic-modules/module-list

    the ModuleList module shows the list of @magic-modules you are looking at.
    docs / demo

  • @magic-modules/theme-list

    the ThemeList module shows a list of all @magic-themes.
    docs / demo

  • @magic-modules/client-list

    the ClientList module shows a list of all @magic-client libraries.
    docs / demo

  • @magic-modules/pre

    the Pre module allows you to display javascript code with syntax highlighting.
    docs / demo

  • @magic-modules/languageswitch

    file hierarchy based multilanguage support for magic.
    docs / demo

  • @magic-modules/video-embed

    the VideoEmbed module embeds videos from any video hoster using iframes. comes with default support for vimeo and youtube.
    docs / demo

  • @magic-modules/gitbadges

    shows a list of github repository status badges.
    docs / demo

@magic-themes

@magic-themes are css-in-js files prepared for usage in @magic. themes allow easy customization of your @magic app

  • @magic-themes/docs

    the @magic-themes/docs theme is used in all @magic documentation pages.
    docs / demo

@magic-client

@magic-client libraries are scripts that provide maximal functionality with minimal footprint.

  • @magic-client/json

    the @magic-client/json module parses and stringifies json. it will also return errors instead of throwing them.
    docs / demo

  • @magic-client/is

    the @magic-client/is module unifies the javascript type testing apis.
    docs / demo
